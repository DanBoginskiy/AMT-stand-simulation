# AMT Stand — симуляція автоматичної транскрипції музики

Інтерактивний навчальний стенд, що **наочно показує весь конвеєр Automatic Music
Transcription (AMT)** у браузері: від синтезованої сирої звукової хвилі →
через спектр, гармоніки, онсети та оцінку висоти → до нот, MIDI-файлу й
партитури. Двомовний інтерфейс (українська / English).

Ключова ідея: ми **самі синтезуємо** відому мелодію (адитивний синтез:
основний тон + гармоніки + ADSR), тому маємо еталон і на кожному кроці бачимо,
наскільки точно система відновлює ноти.

## Вимоги / Requirements

- **Node.js** ≥ 18 (рекомендовано 20/22 LTS), **npm** ≥ 9
- Сучасний браузер з підтримкою Web Audio API, Canvas 2D та SVG (Chrome / Edge / Firefox / Safari)

Повний перелік — у [REQUIREMENTS.md](REQUIREMENTS.md). Машиночитний маніфест — `package.json`.

## Запуск / Run

```bash
npm install
npm run dev
```

Відкриється `http://localhost:5173`. Натисніть **«Прогнати конвеєр»** і кроками
пройдіть усі 12 етапів через ліву панель.

Збірка статичної версії: `npm run build` → `dist/` (перегляд: `npm run preview`).

## 12 етапів конвеєра

1. **Еталонна партитура** — відомий набір нот (ground truth).
2. **Адитивний синтез → сира хвиля** — нота = Σ гармонік × ADSR + шум.
3. **Фреймінг + вікно Ганна** — нарізка сигналу на перекривні кадри.
4. **STFT / спектрограма** — час-частотне представлення (+ CQT-стиль log-вид; клік по стовпцю обирає кадр).
5. **Гармонічний аналіз** — піки-партіали та гребінець k·f₀ обраного кадру.
6. **Детекція онсетів** — spectral flux + адаптивний поріг.
7. **Оцінка висоти F0** — YIN / HPS / ACF (моно) або multi-F0 (поліфонія).
8. **Трекінг нот** — збірка покадрових висот у події-ноти (піанорол).
9. **Квантизація** — прив'язка часів до ритмічної сітки, висот до півтонів.
10. **Генерація MIDI** — реальні байти SMF + завантаження `.mid`.
11. **Партитура** — нотний запис (VexFlow).
12. **Точність** — Precision / Recall / F1 проти еталона.

## Керування

- **Вхід:** Моно ⇄ Поліфонія (перемикач перераховує конвеєр).
- **Алгоритм F0:** YIN / HPS / ACF (для моно).
- **Мова:** UA ⇄ EN (без перерахунку).
- **▶ Відтворити:** слухати синтезований звук; playhead синхронно йде по візуалізаціях.

## Архітектура

Чистий браузерний застосунок (Vite + vanilla JS, без бекенду). Уся обробка
сигналу написана прозоро на JS, щоб кожне проміжне значення можна було
візуалізувати:

```
src/
  score/groundTruth.js     етап 1
  synth/                   етап 2 (additive, adsr, noise)
  dsp/                     етапи 3–9,12 (fft, framing, stft, logfreq,
                           harmonics, onset, pitchMono, pitchPoly,
                           noteTracking, quantize, metrics, music)
  export/midiExport.js     етап 10 (ручний запис SMF)
  render/scoreRender.js    етап 11 (VexFlow)
  viz/                     полотна (waveform, spectrogram, spectrum, …)
  pipeline/                run.js (прогін) + stages.js (реєстр панелей)
  ui/                      stepper, controls, playback, dom
  i18n/                    ua.js / en.js
```

Єдина зовнішня залежність — **VexFlow** (рендер нотного стану). FFT, детектори
висоти (YIN/HPS/ACF) і запис MIDI реалізовані вручну для прозорості.
